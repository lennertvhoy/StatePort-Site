01 / What StatePort is
StatePort is a home for ongoing work with AI. You do not start with a blank chat. You open the app for the work you are doing.

02 / What an app keeps together
Each app keeps the work in one place: the project, the conversation, the tools, the decisions, and what happened before.

03 / The basic idea
The idea is simple. Chat helps with the work, but the project is what lasts.

04 / Choose an app
Here I choose a checklist from the catalog. I open it to continue that work.

05 / See the project
The project opens with the things I need to see: what it is, what tools it has, and what needs attention.

06 / Ask inside the work
Now I ask a normal question: what should I do next? The answer stays inside the checklist, next to the work it is talking about.

07 / Plan an action
Next I ask StatePort to plan the next checklist item. This is an action, so it gets more care than a chat reply.

08 / Review before running
Before it can run, I can see what it will do, which tool will run it, and whether it can change the project.

09 / Approve the plan
The plan is ready, but it is waiting. I can read the exact action before I approve it.

10 / Run the check
After I approve it, StatePort runs the check.

11 / Read the result
Here is the result. It finished, and it did not change the project.

12 / Check it later
Later, I can check the request, the approval, and the result together.

13 / Change settings
Settings show what I can change, and what is fixed for this project.

14 / Use another app
Different work can use a different app. This technical workspace has tools for development work.

15 / See the limits
Some tools are not available here. StatePort leaves them turned off instead of pretending they work.

16 / The takeaway
That is the point of StatePort. AI helps inside work you can see, check, and come back to.

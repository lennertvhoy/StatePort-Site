StatePort platform walkthrough — 22 July 2026

01 / Catalog
Start in the catalog. Choose the app you actually need. Here, the checklist is ready, the study app is unavailable, and StatePort shows that difference instead of hiding it.

02 / Project view
Open the checklist and you get a real project view: its name, its backup state, the tools it exposes, and where the source came from.

03 / Conversation
Ask a normal question: what should I do next? The answer comes back in the same conversation, with the project still in view.

04 / Review the task
For a more controlled task, choose an action. Before it runs, StatePort shows the engine, the budget, the data it will read, and whether it can change anything.

05 / Approval
Approval is its own step. You can review the exact task before anything executes.

06 / Result
After execution, the result is readable: verify the portable execution boundary. The project is unchanged.

07 / Evidence
The evidence drawer keeps the lifecycle, receipt, and exact record together. You can inspect the result instead of trusting a screenshot.

08 / Settings
Settings are part of the product. You can see the instance identity, what stays editable, and which facts come from the application itself.

09 / Another application
Different application, different role. This CTO fixture opens a workbench for development work, while the project identity and recovery state stay visible.

10 / Capability boundary
The boundaries are visible too. Tools that are unavailable in this environment stay unavailable instead of pretending to work.

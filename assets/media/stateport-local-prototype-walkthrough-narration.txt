01 / Catalog
This is the list of apps in StatePort. I am opening the checklist.

02 / Project view
This is the checklist's home. It shows the project, the tools it has, and whether the backup needs attention.

03 / Conversation
Here I ask, what should I do next? The answer stays inside the checklist, next to the work it is talking about.

04 / Review the task
Now I pick an action: plan the next checklist item. Before it runs, I can see what it will do, which tool will run it, and whether it can change anything.

05 / Approval
This request is waiting for my approval. I can read the exact action before I allow it to run.

06 / Result
It is finished. The result says the check is complete. Nothing in the project was changed.

07 / Evidence
Later, I can review the request, the approval, and the result in one place.

08 / Settings
These are the settings. They show what I can change, and what the app controls for me.

09 / Another application
Here is a different app for technical work. It has a different workspace and different tools.

10 / Capability boundary
Some tools are not available in this setup. StatePort leaves them turned off instead of pretending they work.

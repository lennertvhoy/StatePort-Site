What is StatePort?

StatePort is an application-first Stateware platform. It is designed so a person interacts with an installed application—its conversation, actions, progress, and state—rather than beginning with a generic agent control panel.

The application’s durable source of truth is intended to remain readable and inspectable. Models, agents, and execution hosts are useful processors around that truth, not the exclusive place where it lives.

What is Stateware?

Stateware is software whose durable, inspectable state and governed lifecycle contracts form the application boundary while agents and models remain replaceable processors.

That boundary matters when a conversation is compacted, a provider changes, an execution host is unavailable, or a person needs to understand why a change happened. The durable records should still make sense.

What is StateSpec?

StateSpec is the portable, repository-native specification used to describe an application’s durable facts, ownership, requested capabilities, lifecycle rules, and validation expectations. StateDD is the former name retained only for compatibility with older identifiers.

A StateSpec template may request what it needs. It does not grant itself permission: effective capability depends on the template request, the instance grant, and operator policy.

What StatePort does not promise here

  • No claim that a template or StateSpec outperforms a baseline without controlled, reproducible evidence.
  • No claim of a public release, license, download, hosted service, or production readiness before one is published.
  • No claim that a model session, a generated summary, or a screenshot is canonical application truth on its own.

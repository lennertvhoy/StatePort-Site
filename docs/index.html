What is StatePort?

StatePort is a harness that wraps coding agents like Codex and runs them headlessly in environments you control — your local machine or an isolated container. It orchestrates the agent, governs its risky steps, and writes the result into durable state you own.

Each application is a Stateware template: durable state files (Markdown and YAML) plus the cockpit scripts the agent is allowed to run. The agent is a replaceable processor; the work stays yours.

What is Stateware?

Stateware is our name for software that keeps the important part of an application in a place people can read, own, and carry forward.

Chat and AI tools can help, but they should not be the only place your memory lives or the only way to understand what changed.

What is StateSpec?

StateSpec is the format used to describe an application: what it is for, what it needs to remember, and what it is allowed to do. You do not need to know this format to understand the product.

The deeper pages explain how StateSpec keeps permissions, changes, and ownership clear. StateDD is its former name, kept only for older files and tools.

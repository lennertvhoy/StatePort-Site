What the video shows

  1. Application-first home. Work resumes in a named application rather than an anonymous chat thread.
  2. Visible source context. The screen keeps the fixture and source status in view.
  3. Local conversation path. One concise prompt receives a durable reply through a locally authenticated command-line session.
  4. Optional Workbench. Technical capabilities appear when they are useful and show their current state.
  5. Source boundary. An exact development candidate can support isolated testing while release material remains separately identified.
  6. Mobile home. The installed-applications orientation survives a narrow viewport without making the Workbench the default product.

Where the video stops

This is a local capture. Packaging, fresh-machine setup, host coverage, benchmarks, and independent checks each need their own record. The evidence ladder explains how those records fit together.

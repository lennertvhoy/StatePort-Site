1 minute 44 second captioned walkthrough recorded from the working public-safe fixture.

What you will see

  1. Start with the idea. StatePort keeps a conversation with the work instead of treating chat as the work.
  2. Choose an application. The catalog lets a person continue a checklist or open a different workspace.
  3. Ask inside the work. A normal question receives an answer alongside the checklist it concerns.
  4. Review an action. Before a task runs, the exact action, what runs it, and its ability to change the project are visible.
  5. Approve, run, and read the result. The fixture shows a reviewed action, an execution, and a result that did not change the project.
  6. Return later. Evidence, settings, and a second app show what happened, what can change, and what is unavailable.

How to read the claims

The video shows a working public-safe fixture and its recorded UI results. It does not turn that fixture into a downloadable software release; the release ledger and proof and roadmap keep those claims separate.

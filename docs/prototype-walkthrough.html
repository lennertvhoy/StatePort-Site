What you will see

  1. Choose an application. The catalog shows availability and installed instances.
  2. Ask a normal question. The answer stays in the project conversation.
  3. Review before running. The task, engine, budget, and data boundary are visible first.
  4. See the result. The run returns a readable answer and says whether the project changed.
  5. Inspect evidence and settings. Receipts, identity, recovery, and capability boundaries remain available.

How to read the claims

The video shows a working public-safe fixture and its recorded UI results. It does not turn that fixture into a downloadable software release; the release ledger and proof and roadmap keep those claims separate.
